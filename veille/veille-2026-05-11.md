## 🔒 Cybersécurité IA

### Vulnérabilités critiques — Semantic Kernel (divulgation du 7 mai)

**CVE-2026-26030 — RCE via injection de prompt (Python SDK)**
- **Composant vulnérable** : In-Memory Vector Store avec filtre par défaut
- **Mécanisme** : le filtre lambda Python exécuté via `eval()` interpole directement la valeur `city` fournie par l'utilisateur — une seule invite suffit à lancer un processus arbitraire (ex. : `calc.exe`)
- **Patch** : Python `semantic-kernel >= 1.39.4` — correction via AST node-type allowlist + function call allowlist + name node restriction
- **Sévérité** : Critique (CVSS NVD à confirmer)

**CVE-2026-25592 — Écriture de fichier arbitraire (.NET SDK)**
- **Composant vulnérable** : `SessionsPythonPlugin.DownloadFileAsync` exposé par erreur au modèle
- **Mécanisme** : le paramètre `localFilePath` sans validation de chemin permet d'écrire des fichiers n'importe où sur l'hôte
- **Patch** : .NET SDK `>= 1.71.0`

**Microsoft Agent Governance Toolkit** : en réponse à ces failles, Microsoft a publié un toolkit open-source dédié à la sécurité runtime des agents, incluant sandbox de permissions et journaux d'audit. [(Microsoft Open Source)](https://opensource.microsoft.com/blog/2026/04/02/introducing-the-agent-governance-toolkit-open-source-runtime-security-for-ai-agents/)

### Contexte élargi

- Le blog Microsoft Security (7/05) montre que la connexion prompt injection → RCE est désormais réelle et documentée sur des frameworks en production — ne nécessite aucune exploitation mémoire classique.
- Le rapport ArXiv 2506.23260 dresse une taxonomie des menaces dans les workflows d'agents LLM (injections de prompt, exploits de protocole, détournement d'outil), publié en mai 2026. [(arXiv)](https://arxiv.org/html/2506.23260v1)
