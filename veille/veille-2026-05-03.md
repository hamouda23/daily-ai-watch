## 🔒 Cybersécurité IA

### Recherche & publications

- **PCAS (ICLR 2026) — Compilateur de politiques pour agents sécurisés** — Le *Policy Compiler for Secure Agentic Systems* compile les implémentations d'agents avec des spécifications de politique formelles pour produire des systèmes respectant automatiquement les règles métier. Résultats clés : conformité aux politiques passée de 48 % à 93 % sur plusieurs modèles frontier (Claude, GPT, Gemini), zéro violation dans les runs instrumentés. Un résultat particulièrement pertinent pour les systèmes RAG agentiques déployés en production. [(ArXiv — PCAS)](https://arxiv.org/html/2602.16708v1)

- **Étude ArXiv — 94,4 % des LLMs vulnérables à l'injection directe de prompts** — Une évaluation de 18 LLMs état de l'art publiée sur ArXiv révèle que 94,4 % succombent à l'injection directe de prompts et que 83,3 % sont vulnérables aux attaques RAG Backdoor. Ces chiffres montrent que malgré les efforts de durcissement, la surface d'attaque reste très large, notamment via les vecteurs multimodaux (instructions malveillantes encodées dans des images par perturbations adversariales invisibles). [(ArXiv — Security Survey)](https://arxiv.org/abs/2504.19678)

- **Expansion de la surface d'attaque via MCP (Model Context Protocol)** — La généralisation du MCP pour orchestrer des agents LLM introduit de nouveaux vecteurs critiques : empoisonnement d'outils (*tool poisoning*), vol de credentials, exécution de code arbitraire. Des chercheurs documentent ces risques dans plusieurs papers ICLR 2026, pointant l'absence de sandboxing robuste dans la majorité des frameworks agentiques actuels. [(ICLR 2026 — Lambda AI Blog)](https://lambda.ai/blog/iclr-2026-12-papers)
